# TF1-synthetic corpus build — design spec

**Branch:** `tf1-iou-diagnostic`
**Date:** 2026-05-06
**Related:** [`experiments/11_tf1_diagnostic/REPORT.md`](../../../experiments/11_tf1_diagnostic/REPORT.md)

## Goal

Produce a MORABLES-shaped derivative of the TF1-EN-3M synthetic dataset under `data/external/tf1_synthetic/` so we can later use it in retrieval training and evaluation experiments with the same code paths as the curated MORABLES corpus.

This spec covers the **data layout + two build scripts** only. The downstream step — formatting the data as `(anchor, positive, negatives[])` triples with sibling-aware negative sampling — is a separate spec (parking-lot at the bottom).

## Non-goals

- Not modifying any file under `data/processed/` or `data/clustered/` (MORABLES is left alone).
- Not running training on this data yet.
- Not human-reviewing the TF1 moral clusters (MORABLES used a CLI review tool; we skip it because we only have 100 morals to inspect — manual eyeballing of `cluster_mapping.json` is sufficient).
- Not adding TF1 morals to MORABLES splits or any qrels under `data/processed/`.

## Inputs

| Input | Path | Source |
|---|---|---|
| TF1 sample dump | `experiments/11_tf1_diagnostic/results/runs/<latest>/samples.jsonl` | Produced by [`check_iou.py`](../../../experiments/11_tf1_diagnostic/check_iou.py); 50K rows with `idx, chunk, prompt_hash, moral, fable, …` — already stratified across the 3M dataset |
| 100 unique morals (derived) | Implicit — first occurrence of each unique `moral.lower().strip()` in the dump | Stable ordering by first-seen index |

We do **not** re-stream HF in the build scripts. If a future N > 500 is needed (the cache supports up to ~500 per moral), we'll re-run `check_iou.py --n <bigger>` first and re-point the build script at the new run dir.

## Outputs — directory layout

```
data/external/tf1_synthetic/
  README.md                                       # provenance: HF id, paper, build params, command history

  raw/                                            # gitignored (placeholder for any future stream cache)

  processed/                                      # mirrors data/processed/
    fables_corpus.json                            # 100 × N entries
    morals_corpus.json                            # 100 entries
    qrels_moral_to_fable.json                     # 100 × N rows (1:N)
    qrels_fable_to_moral.json                     # 100 × N rows (1:1)

  clustered/                                      # mirrors data/clustered/
    morals_unique_corpus.json                     # K cluster reps
    cluster_mapping.json                          # K cluster definitions
    moral_to_cluster.json                         # 100 → K mapping
    qrels_moral_to_fable_clustered.json           # 1:M
    qrels_fable_to_moral_clustered.json           # 1:1 (each fable maps to its cluster rep)
```

The `data/external/tf1_synthetic/raw/` exclusion gets appended to `data/external/.gitignore` (alongside the existing `storal/raw/` entry).

### File schemas

**`processed/morals_corpus.json`** — list of:
```json
{ "doc_id": "moral_tf1_000", "text": "greed leads to downfall" }
```
The `_tf1_` infix keeps these globally distinct from MORABLES' `moral_0000`.

**`processed/fables_corpus.json`** — list of:
```json
{
  "doc_id": "fable_tf1_00000",
  "text": "<full fable text>",
  "moral_id": "moral_tf1_000",
  "source_idx": 12345,
  "source_chunk": 2,
  "prompt_hash": "<sha256 from HF row>"
}
```
`source_idx` and `source_chunk` let us trace each entry back to the row in `samples.jsonl` it came from. `prompt_hash` is TF1's native row identifier (useful for dedup if we ever combine multiple runs).

**`processed/qrels_moral_to_fable.json`** — list of:
```json
{ "query_id": "moral_tf1_000", "doc_id": "fable_tf1_00000", "relevance": 1 }
```
100 × N rows (one per moral-fable pair).

**`processed/qrels_fable_to_moral.json`** — inverse:
```json
{ "query_id": "fable_tf1_00000", "doc_id": "moral_tf1_000", "relevance": 1 }
```
100 × N rows (each fable has exactly one ground-truth moral).

**`clustered/morals_unique_corpus.json`** — list of:
```json
{
  "doc_id": "moral_tf1_unique_0000",
  "text": "<canonical text — first member of the cluster>",
  "cluster_id": "<auto-generated, e.g. 'cluster_007'>",
  "cluster_type": "singleton" | "near" | "exact",
  "relevant_fable_ids": ["fable_tf1_00012", ...],
  "cluster_moral_set": ["honesty is the best policy", "honesty outlives deceit", ...]
}
```
Mirrors MORABLES' `morals_unique_corpus.json` schema exactly so downstream consumers can be reused.

**`clustered/cluster_mapping.json`** — list of:
```json
{
  "cluster_id": "cluster_007",
  "type": "near",
  "moral_set": ["honesty is the best policy", "honesty outlives deceit"],
  "fables": ["fable_tf1_00012", "fable_tf1_00013", ...],
  "n_morals": 2,
  "n_fables": 20
}
```

**`clustered/moral_to_cluster.json`** — list of:
```json
{ "query_id": "moral_tf1_000", "text": "greed leads to downfall", "cluster_id": "cluster_003" }
```

**`clustered/qrels_moral_to_fable_clustered.json`** — list of:
```json
{ "query_id": "moral_tf1_unique_0000", "doc_id": "fable_tf1_00012", "relevance": 1 }
```
All fables belonging to any cluster member moral.

**`clustered/qrels_fable_to_moral_clustered.json`** — list of:
```json
{ "query_id": "fable_tf1_00012", "doc_id": "moral_tf1_unique_0000", "relevance": 1 }
```
Each fable → its cluster's canonical query id.

## Scripts

### `experiments/11_tf1_diagnostic/build_tf1_corpus.py`

**Args:**
- `--n <int>` — fables per moral (default **10**, max 500 from cache)
- `--source <path>` — path to a `samples.jsonl` (default: latest under `experiments/11_tf1_diagnostic/results/runs/`)
- `--seed <int>` — RNG seed for fable sampling (default 42)
- `--out <path>` — output dir (default `data/external/tf1_synthetic/`)

**Behavior:**
1. Resolve `--source` (auto-pick latest run dir if not provided).
2. Stream samples.jsonl, group rows by `moral.lower().strip()`.
3. Assign stable moral ids in first-seen order → `moral_tf1_000` … `moral_tf1_099`.
4. For each moral group, sample `N` rows with `random.Random(seed).sample(...)`. Fail loudly (clear error message) if a moral has fewer than N samples cached.
5. Assign stable fable ids `fable_tf1_00000` … `fable_tf1_NNNNN` in (moral_idx × N + i) order so reruns with the same seed produce byte-identical output.
6. Emit the 4 `processed/` files + `README.md`.
7. Print a summary: N per moral, total fables, output paths, args used.

**Validation:**
- Assert exactly 100 unique morals discovered (the known invariant from the diagnostic).
- Assert each moral gets exactly N fables.
- Assert no duplicate `prompt_hash` across the chosen fables.

### `experiments/11_tf1_diagnostic/cluster_tf1_morals.py`

**Args:**
- `--threshold <float>` — primary cosine threshold for cluster merging (default **0.80**, matching MORABLES' candidate threshold `SIM_THRESHOLD_MED`)
- `--inspect-thresholds <comma-separated floats>` — extra thresholds to dump for comparison (default `"0.80,0.85,0.90"`)
- `--model <str>` — sentence embedder (default `BAAI/bge-large-en-v1.5`, same as `analysis/moral_similarity.py`)
- `--in <path>` — input dir (default `data/external/tf1_synthetic/`)

**Behavior:**
1. Load `processed/morals_corpus.json` (100 morals) and `processed/qrels_moral_to_fable.json`.
2. Embed all 100 morals once with the chosen sentence-transformer model.
3. Build the 100×100 cosine similarity matrix.
4. For each threshold in `--inspect-thresholds`, run agglomerative clustering (`scipy.cluster.hierarchy` or equivalent) with `1 − threshold` distance cutoff and dump a small `experiments/11_tf1_diagnostic/cluster_inspection/<timestamp>/clusters_at_<t>.json` showing the resulting partition. (Inspection dumps live alongside the experiment, not in `data/external/`, to keep the corpus dir clean of transient analysis.)
5. Use the `--threshold` partition as the canonical output. Emit the 5 `clustered/` files.
6. Classify each cluster as `exact` (≥2 morals, ≥1 pair with cosine 1.0), `near` (≥2 morals, no exact pair), `singleton` (1 moral).
7. The canonical text per cluster = the cluster member with the **highest total occurrence count** in `samples.jsonl` (most-frequent paraphrase). Tie-break by first-seen order.
8. Print summary: K clusters, type breakdown, max cluster size, the 5 largest cluster contents.

**Validation:**
- Assert every moral appears in exactly one cluster.
- Assert union of cluster fable-sets equals the full fable-set in `processed/`.
- Assert `qrels_fable_to_moral_clustered` has exactly one row per fable (single positive).

## Reproduction

```bash
# Assumes results/runs/<latest>/samples.jsonl exists (50K rows from the diagnostic)
./run.sh experiments/11_tf1_diagnostic/build_tf1_corpus.py --n 10
./run.sh experiments/11_tf1_diagnostic/cluster_tf1_morals.py --threshold 0.80
```

## Risks / open issues

- **Threshold 0.80 is the MORABLES *candidate* threshold, not its final partition** — MORABLES went through human review afterward. For TF1 we're auto-finalizing, so the resulting K will probably be lower than what a human reviewer would set. Mitigation: the side-by-side `0.80 / 0.85 / 0.90` dump lets us re-pick after eyeballing.
- **N=10 is small relative to the 30K-per-moral pool**, but matches the "MORABLES has ~1 fable per moral × 10×" intuition. Easy to re-run with a larger N if we want to scale.
- **No re-streaming** — the cache caps N at ~500/moral. We accept this for now.
